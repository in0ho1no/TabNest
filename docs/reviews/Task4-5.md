## 実装上の判断(レビュアー承認済み)

- どの列のソートでも「フォルダを先頭に表示」(SPEC 初期ソートの方針)を不変条件として維持する
- 列の区切りは透明 Button(最小 ControlTemplate・IsTabStop=False)。Rectangle 等の Shape は
  UIA から検出できず UI テストで操作不能なため
- ソート方向インジケータ(▲▼)は SPEC に記載がないため追加しない(仕様外拡張の回避)

## 確認済み項目(レビュアー所見)

- 層分離: ソートロジック(ToggleSort/ApplySort)・クランプ計算(ClampAutoColumnWidth)は
  ViewModels 層で WinUI 非依存。View は文字列測定と幅適用のみ
- 仮想化対応: 自動調整時に実体化済みの行へ適用し、ContainerContentChanging で後から
  実体化する行にも適用
- ソート設定はフォルダ移動後も維持(LoadFolderCore が ApplySort を通る)
- ダブルクォート: 両側揃いのみ除去。片側のみ・空(`""`)はエラーで状態不変。境界も適切
- テスト14件(ソート列別・昇降順・フォルダ先頭維持・クランプ境界・クォート3種)で実装を実証
